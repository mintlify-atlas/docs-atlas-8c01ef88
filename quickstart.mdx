This guide will walk you through creating your first project and workspace with GitSpace.

## Launch the TUI

The easiest way to use GitSpace is through its interactive Terminal User Interface (TUI):

```bash
gssh
```

The TUI provides a two-panel interface:
- **Left panel**: Your projects
- **Right panel**: Workspaces in the selected project

### Key Bindings

| Key | Action |
|-----|--------|
| `Enter` | Select project / Open workspace |
| `Tab` | Switch between panels |
| `n` | New project / workspace |
| `d` | Delete selected item |
| `?` | Show help |
| `q` | Quit |

## Create Your First Project

<Steps>
  <Step title="Add a project">
    You can add a project either through the TUI or CLI:

    <CodeGroup>
    ```bash TUI
    # Launch TUI and press 'n' to create a new project
    gssh
    ```

    ```bash CLI
    # Use the CLI directly
    gssh project add
    ```
    </CodeGroup>

    You'll be prompted to choose between:
    - **Enter git remote URL** - Clone from any git repository
    - **Choose GitHub repository** - Select from your GitHub repositories
  </Step>

  <Step title="Select a repository">
    If you chose "Choose GitHub repository", GitSpace will fetch your repositories:

    ```bash
    Fetching repositories...
    ? Select a repository: (Use arrow keys)
    > myorg/awesome-project
      myorg/another-project
      personal/side-project
    ```

    <Info>
    Use the `--org` flag to filter repositories by organization:
    ```bash
    gssh project add --org myorg
    ```
    </Info>
  </Step>

  <Step title="Clone and configure">
    GitSpace will automatically:
    - Clone the repository to `~/gitspace/<project-name>/base`
    - Detect the default branch (usually `main` or `master`)
    - Run onboarding steps if a bundle is present
    - Create project configuration

    ```bash
    ✓ Cloning repository...
    ✓ Detecting default branch: main
    ✓ Project 'awesome-project' created successfully
    ```
  </Step>
</Steps>

## Create Your First Workspace

Workspaces are isolated git worktrees for working on different features or branches simultaneously.

<Steps>
  <Step title="Create a workspace">
    <CodeGroup>
    ```bash TUI
    # In TUI, select project and press 'n' in right panel
    gssh
    ```

    ```bash CLI
    # Create workspace with custom name
    gssh workspace add my-feature --project awesome-project
    ```
    </CodeGroup>

    <Note>
    If you have Linear integration configured, you can create workspaces directly from Linear issues.
    </Note>
  </Step>

  <Step title="Enter workspace details">
    You'll be prompted for:
    - **Workspace name** - Will be used for the directory and branch
    - **Branch name** (optional) - Defaults to workspace name
    - **Base branch** (optional) - Defaults to project's main branch

    ```bash
    ? Workspace name: feature/user-authentication
    ? Create from branch: (main)
    ```
  </Step>

  <Step title="Workspace creation">
    GitSpace will:
    - Create a new git worktree at `~/gitspace/<project>/workspaces/<workspace-name>/`
    - Create and checkout a new branch
    - Run setup scripts (if configured in `.gitspace/scripts/setup/`)
    - Open the workspace in a new terminal session

    ```bash
    ✓ Creating worktree...
    ✓ Checking out branch: feature/user-authentication
    ✓ Running setup scripts...
    ✓ Workspace ready!
    ```
  </Step>
</Steps>

## Working with Workspaces

### Open a Workspace

In the TUI, select a workspace and press `Enter` to open it in a new terminal session.

```bash
# Or use the CLI to list workspaces
gssh workspace list --project awesome-project
```

**Example output:**

```bash
Workspaces in 'awesome-project':

  feature/user-authentication    (branch: feature/user-authentication)
  bugfix/login-error            (branch: bugfix/login-error)
  main-workspace                (branch: main)
```

### Workspace Session Mode

When you open a workspace, GitSpace injects a `space` shell function that provides workspace-scoped commands:

```bash
# Show workspace context
space context

# Show workspace context as JSON
space context --json
```

<Warning>
Inside a workspace session, regular `gssh` commands are restricted to prevent cross-workspace mistakes.
</Warning>

### View Workspace Status

Get detailed information about a workspace:

```bash
gssh workspace context --project awesome-project --workspace feature/user-authentication
```

**Example output:**

```json
{
  "project": "awesome-project",
  "workspace": "feature/user-authentication",
  "branch": "feature/user-authentication",
  "path": "/Users/you/gitspace/awesome-project/workspaces/feature/user-authentication",
  "hasUncommittedChanges": false,
  "lastModified": "2026-03-01T10:30:00Z"
}
```

## Common CLI Commands

### Project Management

```bash
# List all projects
gssh project list

# List projects with details
gssh project list --verbose --json

# Remove a project
gssh project remove awesome-project
```

### Workspace Management

```bash
# List workspaces in a project
gssh workspace list --project awesome-project

# Create workspace from specific branch
gssh workspace add hotfix --project awesome-project --from production

# Create workspace with different branch name
gssh workspace add my-workspace --project awesome-project --branch feature/custom-name

# Skip setup scripts
gssh workspace add quick-test --project awesome-project --no-setup

# Remove a workspace
gssh workspace remove feature/user-authentication --project awesome-project
```

## Directory Structure

After creating a project and workspace, your directory structure will look like:

```bash
~/gitspace/
├── .config.json                          # Global configuration
├── awesome-project/
│   ├── .config.json                      # Project configuration
│   ├── base/                             # Base repository clone
│   │   └── ...                           # Your repo files
│   └── workspaces/
│       └── feature/user-authentication/  # Workspace (git worktree)
│           ├── .gitspace/                # Bundle scripts (if configured)
│           │   └── scripts/
│           │       ├── setup/            # Run during workspace setup
│           │       ├── select/           # Run when opening workspace
│           │       └── remove/           # Run before deletion
│           ├── gitspace.lock             # Setup completion marker
│           └── ...                       # Your branch files
```

## Using Repo Config Bundles

<Info>
Repo config bundles allow teams to share onboarding configurations, setup scripts, and environment requirements.
</Info>

If your repository contains a `.gitspace/` directory with a `bundle.json` file, GitSpace will automatically:

1. Detect the bundle during project creation
2. Run onboarding steps (verify tools, collect secrets, etc.)
3. Install scripts in each workspace
4. Execute setup scripts when creating workspaces

**Example bundle structure:**

```
.gitspace/
├── bundle.json              # Onboarding configuration
└── scripts/
    ├── setup/
    │   └── 01-install-deps.sh
    ├── select/
    │   └── 01-status.sh
    └── remove/
        └── 01-cleanup.sh
```

### Loading External Bundles

You can also load bundles from external sources:

```bash
# From local directory
gssh project add --bundle-path /path/to/bundle/

# From remote URL
gssh project add --bundle-url https://example.com/bundle.zip

# Skip bundle detection
gssh project add --skip-bundle
```
