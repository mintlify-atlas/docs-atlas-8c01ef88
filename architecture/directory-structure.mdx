This page documents the complete directory structure of the GitSpace CLI codebase.

## Source Code Overview

```
src/
├── index.ts                    # Entry point (TUI or CLI dispatch)
├── app.tui.tsx                 # TUI application (OpenTUI)
├── app.web.tsx                 # Web application (React)
├── commands/                   # CLI command implementations
├── core/                       # Core business logic
├── lib/                        # Shared libraries
├── relay/                      # Relay server
├── serve/                      # Machine daemon
├── session/                    # Session management
├── types/                      # TypeScript type definitions
├── utils/                      # Utility functions
├── tui/                        # TUI-specific components
├── components/                 # Shared UI components
├── hooks/                      # React/TUI hooks
└── cli/                        # CLI utilities
```

## Libraries

### tmux-lite Library

The `src/lib/tmux-lite/` directory implements the terminal multiplexer:

```
lib/tmux-lite/
├── server.ts               # Session manager with xterm-headless
├── cli.ts                  # CLI interface for tmux-lite
├── protocol.ts             # Frame format (PTY/CONTROL)
├── handshake-handler.ts    # X3DH handshake management
├── relay-client.ts         # WebSocket client for relay
└── crypto/                 # Cryptography implementations
    ├── identity.ts         # Ed25519/X25519 key generation
    ├── keyexchange.ts      # X25519 ECDH
    ├── secretbox.ts        # AES-256-GCM encryption
    ├── frames.ts           # Encrypted frame format
    ├── handshake.ts        # X3DH protocol steps
    ├── device-cert.ts      # Device certificate verification
    ├── user-identity.ts    # User root identity (BIP39)
    ├── root-invites.ts     # Signed root invite tokens
    ├── access-control.ts   # ACL checking
    ├── keys.ts             # Key derivation utilities
    └── index.ts            # Crypto module exports
```

<Note>
The crypto implementations are designed to be portable between Node.js and browser environments.
</Note>

  </Tab>

### Protocol Message Types

The relay protocol supports these message categories:

1. **Machine → Relay**: `register_machine`, `unlock_request`, `data`
2. **Client → Relay**: `unlock_relay`, `list_machines`, `connect_to_machine`, `create_root_invite`, `owner_sync_*`
3. **Relay → Machine**: `relay_identity`, `challenge`, `registered`, `client_connected`, `data`
4. **Relay → Client**: `machine_list`, `connection_established`, `root_invite_created`, `owner_sync_*_result`

## Shared Components

The `src/components/` directory contains cross-platform UI components:

### Component Pattern

Each component follows a three-file pattern:

```
components/
├── MachineList.tsx         # Logic + hooks
├── MachineList.web.tsx     # Web rendering (React DOM)
├── MachineList.tui.tsx     # TUI rendering (OpenTUI)
├── SpacesBrowser.tsx       # Workspace browser logic
├── SpacesBrowser.web.tsx
├── SpacesBrowser.tui.tsx
├── Inbox.tsx               # Notification system
├── Inbox.web.tsx
├── Inbox.tui.tsx
├── Flow.tsx                # Modal dialog system
├── Flow.web.tsx
├── Flow.tui.tsx
├── ProjectList.tsx
├── ProjectList.web.tsx
└── ProjectList.tui.tsx
```

<Accordion title="Why Three Files?">
This pattern allows:
- **Shared business logic** - Common hooks and state management
- **Platform-specific rendering** - Optimized for TUI or web
- **Type safety** - TypeScript enforces consistent interfaces
- **Code reuse** - Logic written once, rendered twice
</Accordion>

<CodeGroup>
```typescript Identity Types

  id: string;

}
```

## File System Layout

GitSpace uses a well-defined file system structure:

### User Data Directory

```
~/gitspace/
├── .config.json            # Global configuration
├── .identity/              # User and machine identities
│   ├── keypair.json        # Encrypted identity keypair
│   ├── machine.json        # Machine registration info
│   └── relay.json          # Relay configuration cache
├── .relay/                 # Relay server data
│   └── control/
│       └── control.db      # ACL + invite state (SQLite)
├── .serve/                 # Machine daemon state
│   ├── serve.pid           # Daemon PID file
│   └── status.sock         # Status socket
├── host.json               # gitspace.sh subdomain config
└── <project-name>/         # Project directories
    ├── .git/               # Base repository
    ├── .config.json        # Project config
    └── workspaces/         # Workspace worktrees
        └── <workspace>/
            ├── .git        # Worktree link
            ├── .gitspace/  # Workspace config
            └── gitspace.lock # State marker
```

### Temporary Data

```
/tmp/
└── tmux-lite-sessions/     # Session data (configurable)
    └── <session-id>/
        ├── state.json      # Session state
        └── buffer          # Terminal buffer
```

<Note>
The tmux-lite session directory can be configured via the `TMUX_LITE_SESSION_DIR` environment variable.
</Note>

## Configuration Files

### Global Config

**Location**: `~/gitspace/.config.json`

```json
{
  "currentProject": "my-project",
  "baseBranch": "main",
  "staleWorkspaceDays": 30
}
```

### Project Config

**Location**: `~/gitspace/<project>/.config.json`

```json
{
  "name": "my-project",
  "repository": "git@github.com:user/repo.git",
  "baseBranch": "main",
  "workspaces": [
    {
      "name": "feature-login",
      "branch": "feature/login",
      "createdAt": "2024-01-15T10:00:00Z"
    }
  ]
}
```

### Environment Variables

| Variable | Description | Default |
|----------|-------------|----------|
| `RELAY_PORT` | Relay server port | `4480` |
| `RELAY_BIND` | Relay bind address | `0.0.0.0` |
| `RELAY_PRIVATE_KEY` | Base64 Ed25519 private key | Uses keychain |
| `RELAY_LABEL` | Label for relay identity | None |
| `GITSPACE_API_URL` | gitspace.sh API URL | `https://api.gitspace.sh` |
| `TMUX_LITE_SESSION_DIR` | Session data directory | `/tmp/tmux-lite-sessions` |
