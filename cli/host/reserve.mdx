## Overview

Reserve a subdomain on `gitspace.sh` to enable remote terminal access without port forwarding or VPN. Your subdomain provides:

- **Primary subdomain:** `<name>.gitspace.sh` for relay server
- **Wildcard DNS:** `*.<name>.gitspace.sh` for multi-tenant hosting
- **Serve subdomain:** `<name>.serve.gitspace.sh` for machine daemon
- **Cloudflare tunnel:** Automatic HTTPS termination and routing

## Usage

```bash
gssh user host reserve <subdomain>
```

## Arguments

<ParamField path="subdomain" type="string" required>
  Your desired subdomain name. Must be:
  - Lowercase letters, numbers, and hyphens only
  - Between 3-32 characters
  - Not already taken
  - Not a reserved word

  Example: `brad`, `my-workspace`, `dev-team`
</ParamField>

## Examples

<CodeGroup>
```bash Reserve subdomain
gssh user host reserve brad
```


## What Gets Created

### Primary Subdomain

`<name>.gitspace.sh` is used for:
- Relay server (`gssh relay start`)
- WebSocket endpoint: `wss://<name>.gitspace.sh/ws`
- Web interface for remote terminals

### Serve Subdomain

`<name>.serve.gitspace.sh` is used for:
- Machine daemon (`gssh machine serve start`)
- Terminal sessions and workspace access
- PTY multiplexing

### Tunnel Tokens

Secure Cloudflare tunnel tokens are:
- Fetched from gitspace.sh API
- Stored in your system keychain
- Used by `cloudflared` to establish tunnels

**Token storage:**
```
TUNNEL_TOKEN_<subdomain>         # Primary tunnel
TUNNEL_TOKEN_<subdomain>_serve   # Serve tunnel
```

### Local Configuration

Host config stored in `~/.gitspace/host.json`:
```json
{
  "subdomain": "brad",
  "serveSubdomain": "brad.serve",
  "subdomains": ["brad"],
  "createdAt": 1704067200000
}
```

## Subdomain Limits

**Free tier:** 3 subdomains per account

```
2/3 subdomains used (free tier)
```

To release a subdomain:
```bash
gssh user host release <subdomain>
```

## Primary Subdomain

The first subdomain you reserve is automatically set as **primary**. This is used by default when you run:

```bash
gssh machine serve start   # Uses primary subdomain
gssh relay start           # Uses primary subdomain
```

To change the primary:
```bash
gssh user host set-primary <subdomain>
```

## After Reservation

Once reserved, you can:

### Start the relay server
```bash
gssh relay start
```

Output:
```
Using account host brad.gitspace.sh for relay tunnel
Relay listening on ws://brad.gitspace.sh:4480
Public relay URL: wss://brad.gitspace.sh/ws
```

### Start the machine daemon
```bash
gssh machine serve start
```

Output:
```
Serve daemon started with cloudflared tunnel to brad.serve.gitspace.sh
Public URL: https://brad.serve.gitspace.sh
```

### Connect from anywhere
```bash
gssh client connect <machine-id>
```

Or open in browser:
```
https://brad.gitspace.sh
```

## Troubleshooting

### "Subdomain is not available"

The subdomain is already taken or reserved. Try a different name:
```bash
gssh user host reserve brad-workspace
```

### "Not logged in"

You need to authenticate first:
```bash
gssh user auth login
```

### "Identity not found"

Create your user root identity:
```bash
gssh user identity init
```

### "Failed to get tunnel token"

Network or API issue. Check:
1. Internet connection
2. gitspace.sh API status
3. Try again in a few moments
